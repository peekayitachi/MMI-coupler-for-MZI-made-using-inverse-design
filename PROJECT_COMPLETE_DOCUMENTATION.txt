EXECUTIVE SUMMARY
================================================================================

This project implements a machine learning-based inverse design methodology for
silicon photonic MMI-MZI modulators. The goal is to design device geometries
that achieve target circuit performance specifications (ER≥20dB, BW≥40nm, IL≤1dB).

The project evolved through two major phases:
  • Phase 1 (v1): Initial implementation with synthetic metrics
  • Phase 2 (v2): Enhanced dataset with mixed real/synthetic metrics + validation

Current Status: 100 diverse v2 ensemble candidates evaluated and ready for
real physics validation. Top 3 candidates identified for validation.

PHASE 2A: CRITICAL BUG FIXES
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

Bug #1: Wrong Parameter Bounds
  Location: mmi_mzi_project.py lines 2147-2154
  Issue: Clipping bounds didn't match actual dataset ranges
  Before: W_mmi=[0.5-20.0], L_mmi=[10-500], gap=[0.05-3.0], W_io=[0.2-0.8]
  After:  W_mmi=[3.0-12.0], L_mmi=[30-300], gap=[0.15-1.50], W_io=[0.35-0.55]
  Impact: CRITICAL - Caused degenerate model outputs
  Fix: Applied correct bounds in 3 locations

Bug #2: cGAN Hard Clipping Causing Mode Collapse
  Location: cgan_inverse.py lines 377-408
  Issue: Hard clipping enabled gradient saturation → mode collapse
  Solution: Implemented soft tanh-based clipping
    mid = (lo + hi) / 2.0
    scale = (hi - lo) / 2.0
    clipped = mid + scale * np.tanh((value - mid) / scale)
  Result: Preserved gradient flow but collapse still occurred

Bug #3: MDN Scoring Function
  Location: mmi_mzi_project.py lines 2155-2180
  Issue: Scoring didn't encourage diversity
  Solution: Added boundary penalties + center bonuses
    • Penalty for near-boundary values
    • Bonus for center-of-range values
    • Enabled interior-biased sampling

PHASE 2C: MODEL TRAINING
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

1. FORWARD SURROGATE MODEL (MLP)
   Architecture: Input(8) → Dense(256) → Dense(256) → Dense(256) → Output(8)
   Training: 80 epochs, batch size 512, Adam optimizer
   Inputs: [W_mmi, L_mmi, gap, W_io, taper_len, dW_nm, dGap_nm, lambda_nm]
   Outputs: [Re(S11), Im(S11), Re(S21), Im(S21), Re(S12), Im(S12), Re(S22), Im(S22)]
   
   Results:
     • Training MSE: 0.496
     • Test MSE: 0.502
     • Stable convergence
     • Ready for inverse design coupling
   
   Checkpoint: forward_best.pt (533.9 KB)

2. INVERSE MDN MODEL (Mixture Density Network)
   Architecture: Input(6) → Dense(128) → Dense(64) → Mixture(K=8)
   Training: 100 epochs
   Inputs: [ER1, ER2, IL1, IL2, BW1, BW2]
   Outputs: 5-dimensional geometry (μ, σ, π components for K=8 mixtures)
   
   Results:
     • Final NLL: 7.007
     • Mode collapse observed: All samples → single geometry
     • Generated 10 candidates (all degenerate)
   
   Checkpoint: inverse_best.pt (355.7 KB)
   Issue: Mixed real/synthetic training data (40% synthetic) limited diversity learning

3. INVERSE cGAN MODEL (Conditional Generative Adversarial Network)
   Architecture:
     Generator: Linear(100+6=106) → Linear(512) → Linear(1024) → Linear(5)
     Discriminator: Linear(5+6=11) → Linear(512) → Linear(1024) → Linear(1)
   
   Training: 300 epochs (increased from initial 200)
   Condition: 6-dimensional performance metrics
   Noise: 100-dimensional latent vector
   
   Results:
     • Final D_loss: 1.357
     • Final G_loss: 0.734
     • Mode collapse observed: All samples → corner geometry (different from MDN)
     • Generated 200 candidates (all degenerate)
   
   Checkpoint: G_final.pt
   Issue: Soft clipping prevented hard collapse but didn't preserve diversity

PHASE 2D: DIAGNOSIS & ANALYSIS
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

ROOT CAUSE ANALYSIS - Mode Collapse:

Model Learning: Both MDN and cGAN successfully learned modes from training data
Diversity Issue: Models converged to single mode rather than exploring distribution
Evidence:
  • First training (200 epochs cGAN): Collapsed to W_mmi=-833316718925.50 (overflow)
  • Second training (300 epochs + tanh): Collapsed to W_mmi=3.0 (different corner)
  • Third training (improved scoring): Still collapsed

Hypothesis:
  The mixed real/synthetic dataset (40% synthetic) provides limited diversity
  signals during training. Generative models may need:
    • Higher percentage real data
    • More diverse training geometries
    • Different architecture (e.g., VAE, flow-based models)
    • Curriculum learning strategy

Decision: Ensemble Sampling as Fallback
  Since trained models couldn't preserve diversity, implemented deterministic
  ensemble sampling to guarantee design space coverage.

================================================================================
LESSONS LEARNED & INSIGHTS
================================================================================

1. DATA QUALITY MATTERS
   - 40% synthetic data limited generative model diversity
   - Pure trained models couldn't learn distribution adequately
   - Ensemble sampling proved more reliable fallback
   
2. PARAMETER BOUNDS ARE CRITICAL
   - Wrong bounds caused catastrophic failures in first attempt
   - Always verify scaling against training data statistics
   - Applied in multiple code locations → easy to miss one
   
3. MODE COLLAPSE IN GENERATIVE MODELS
   - Hard clipping can cause gradient saturation
   - Soft clipping helps but doesn't prevent collapse if data lacks diversity
   - May need curriculum learning or progressive training
   
4. FORWARD MODEL IL PREDICTIONS
   - Systematic offset of ~2.7 dB from target
   - Could indicate model calibration issue
   - Must validate against real physics before redesign
   
5. ENSEMBLE SAMPLING AS FALLBACK
   - Deterministic sampling guarantees coverage
   - Can be combined with ML for hybrid approach
   - More reliable than 100% ML when models underperform
   
6. DESIGN SPACE EXPLORATION
   - Parameter sensitivity not uniform (W_mmi vs L_mmi)
   - Top performers show W_mmi clustering (not random)
   - Could optimize further with parameter-specific analysis
